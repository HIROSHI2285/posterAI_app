## 🔴 現在の技術的制約

### AIが生成する画像の本質的な制約

1. **ラスター画像（PNG）のみ**
   - すべてのパーツがピクセルに「焼き付け」られている
   - レイヤー情報、テキスト情報が含まれていない
   - パーツの境界を自動認識できない

2. **編集の限界**
   - ❌ テキストを直接編集（文字列として）
   - ❌ パーツを個別に選択・移動
   - ❌ フォント・スタイル情報を抽出
   - ✅ プロンプトでAIに再生成を依頼（ただしスタイルが変わる可能性）

3. **Gemini APIの制約**
   - 入力：テキストプロンプト + 画像
   - 出力：PNG画像のみ
   - SVG、PSD、レイヤー情報は生成不可能

- 反復生成による品質向上
- ユーザーフィードバックの学習

#### アプローチ2：レイヤー編集機能の実現

**技術的なブレークスルーが必要**：

##### オプションA：ハイブリッド生成
```
1. AIが構造定義（JSON）を生成
   {
     background: { type: 'gradient', colors: [...] },
     title: { text: '...', font: '...', position: {...} },
     elements: [...]
   }
   
2. フロントエンドでレンダリング
   Canvas / SVG / Fabric.js
   
3. ユーザーが自由に編集
   
4. PNGとして出力
```

**メリット**：
- ✅ 完全な編集が可能
- ✅ レイヤー情報を保持

**課題**：
- AIが構造定義を生成する機能が必要
- Gemini APIの制約を超える（新しいAPI待ち）

##### オプションB：AI支援型Canvas編集
```
1. AIが背景画像を生成（PNG）
   
2. Canvas UI上に配置
   
3. AIがパーツの位置を推定・提案
   "このテキストはここにあります"
   "このロゴはここです"
   
4. ユーザーが確認・調整
   
5. レイヤー情報として保存
```

**メリット**：
- ✅ 現在の技術で実現可能
- ✅ AIの生成品質を活用

**課題**：
- AIのパーツ推定精度（60-70%）
- ユーザーの手動調整が必要

##### オプションC：次世代AI技術の活用
```
将来的なAI技術（2025年以降？）：
- レイヤー情報付き画像生成
- SVG / PSD 形式での出力
- パーツ単位での編集指示
```

**現状**：
- まだ実現されていない
- 技術進化を待つ必要がある
