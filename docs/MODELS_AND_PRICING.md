## 🎯 各最適化の詳細

### 1. 4K高解像度化（2026-03-09 適用）
`imageSize: '2K'` → `'4K'` に変更。コストは同じ約¥10のまま、出力品質が向上。

### 2. キャラクターSeed永続化（2026-03-09 適用）
`character_features.seed` をレスポンスに含め、フロントエンドが次回生成時に渡すことで同一キャラクターが再現される。コスト影響なし。

### 3. smart-editへのthinkingConfig追加（2026-03-09 適用）
`thinkingBudget: 1024`（medium相当）を設定。分類精度向上でFalse positiveによる不要な画像生成呼び出しを削減。

### 4. Context Caching（in-memoryキャッシュ）
extract-blueprintの解析結果をTTL10分でキャッシュ。同一セッション内の連続解析コストをほぼゼロに。

## 🔧 環境変数による切り替え

```env
# 生成モデル（デフォルト: gemini-3.1-flash-image-preview）
GEMINI_IMAGE_MODEL=gemini-3.1-flash-image-preview

# 編集モデル（デフォルト: gemini-3-pro-image-preview）
GEMINI_EDIT_MODEL=gemini-3-pro-image-preview

# 解析モデル（デフォルト: gemini-3.1-pro-preview）
GEMINI_ANALYSIS_MODEL=gemini-3.1-pro-preview

# Smart Editモデル（デフォルト: gemini-3.1-flash-lite-preview）
GEMINI_SMART_EDIT_MODEL=gemini-3.1-flash-lite-preview
```

再デプロイ不要で即時切り替え可能。
